# Aruco Marker Placer

Application for placing ArUco markers on a drawing/image and exporting the project as PNG/SVG files and a JSON description of the markers. You can place markers by clicking on the image or by entering coordinates manually. It is also possible to save a group of markers as a single image so it can be printed on a plotter without worrying about distances between markers. Further description generated by AI:

## Features
- Load a background image.
- Load a DXF file for automatic scale calibration.
- Select ArUco dictionary, `id`, marker size on the background (`px`), real marker size (`mm`), and rotation (`yaw`).
- Automatic calculation of marker size in pixels (`size_px`) from `size_mm` and `pixelCountPerMeter`.
- Place markers by clicking on the image or by manually entering `x/y` coordinates (in meters).
- Import a layout from a `json` file such as `scheme(5).json`.
- Save the project into a folder containing:
  - `scheme.json`
  - `layout.preview.png`
  - individual marker images: `id<id>_<size_mm>mm.png` and `id<id>_<size_mm>mm.svg`

## DXF + Background Auto Calibration
1. Load a background image.
2. Load a DXF file.
3. Click `Calibrate ppm from DXF`.

What happens:
- The modelspace bounds (`X/Y`) are extracted from the DXF with `INSUNITS` taken into account.
- `pixelCountPerMeter` is calculated using the DXF bounds and the background image size.
- Then `size_px` is automatically recalculated from `size_mm`.

## Coordinates
- Marker coordinates (`x`, `y`) are stored in meters inside the `json`.
- Origin point: center of the background image.
- `x`: positive to the right.
- `y`: positive upward.

## Overall Layout Size
- When `Auto-calculate sizeX/sizeY` is enabled, the size is calculated automatically.
- The maximum of the following is used:
  - metric size of the background image,
  - coverage of all placed markers including their sizes.
- Rounding: upward to `0.01 m`.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Run
```bash
python3 app.py
```

## Editing Markers in the List
- Select a row in the table and click `Load selected into form` (or double-click the row).
- Change the parameters (`id`, dictionary, `x/y`, `size_mm`, `yaw`, and in manual mode also `size_px`).
- Click `Update selected marker`.

## Complex Marker Export
- Multiple rows can be selected in the marker table (`Ctrl/Shift` + click).
- When using `Save project to folder`:
  - selected markers (if there are 2 or more) are saved as a single file: `complex_marker.png` and `complex_marker.svg`;
  - unselected markers are saved individually as before.
- The complex file preserves the real marker sizes and distances between them.
- Empty areas between markers are filled with a solid color:
  - black when `colorInverted=true`,
  - white when `colorInverted=false`.
- The `scheme.json` format remains unchanged.
